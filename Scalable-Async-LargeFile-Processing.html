Scalable File Processing Architecture

Problem

Process large files with audit tracking. Sequential processing would overload Integration Server.

Solution

Designed an event-driven architecture using file splitting, parallel processing, and reconciliation logic.

Architecture Flow

1️⃣ MFT → Work Directory
2️⃣ Initial Service
Insert audit START, split file, publish events
3️⃣ Messaging Layer
4️⃣ Parallel Processing
Process chunks → load Snowflake → publish completion
5️⃣ Reconciliation
Track completion via cache
6️⃣ Final
Update audit as COMPLETED
